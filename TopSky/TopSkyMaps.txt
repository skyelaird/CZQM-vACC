// PURPOSE: Display airspace boundaries that change based on position staffing

// DESIGN PHILOSOPHY:
// - Subtlety over shouting: Boundaries inform, aircraft tags are priority
// - Lighter = "not yours": Uncontrolled/delegated areas lighter than base
// - State visualization: Same geometry, different colors = operational state
// - Non-translucent layers: Higher layers completely overwrite lower layers
//
// COLOR STRATEGY (Shades of Grey):
//   Darkest (base) → Lightest (not controlled)
//   ACTIVE_SECTOR_BASE (80,80,80)       = "Mine, controlled"
//   UNCONTROLLED_STATIC (95,95,95)      = "Class G, no service here"
//   INTERNAL_DELEGATION (100,100,100)   = "Carved out from my airspace"
//   CZQM_CZQX_SPLIT (105,105,105)      = "QM/QX boundary when both online"
//   COLD_NEIGHBOR (110,110,110)         = "Exists but unstaffed"
//   HOT_NEIGHBOR (140,140,140)          = "LIVE neighbor - coordinate!"
//
// LAYER STRATEGY:
//   Layer 0: Static uncontrolled areas (always visible)
//   Layer 1: Cold neighbor boundaries (always visible, light grey)
//   Layer 2: Internal delegations (APP/TWR/DG/CB when online)
//   Layer 3: Hot neighbors (overwrites Layer 1 when neighbor online)
//   Layer 4: CZQM/CZQX split (only when both online)
//
// REFERENCE FILES:
//   ESE: CZQQ-DO-NOT-USE_20251107023304-251101-0017.ese
//   DAH: DAH_en_20250807.pdf
//   Charts: LO_07 (Goose area), LO_08 (North)
//   Airspace Manual: CZQM FIR Centre Airspace Manual v2 draft 3.pdf
//
// ============================================================================

// ----------------------------------------------------------------------------
// EXAMPLE: TWR BOUNDARIES USING COORD_CIRCLE
// ----------------------------------------------------------------------------
// These demonstrate the COORD_CIRCLE syntax for simple circular boundaries
// Uncomment and add activation logic as Layer 2 development progresses
//
// SYNTAX: COORD_CIRCLE:AirportCode:RadiusNM:SpacingDegrees
//   - AirportCode: ICAO airport identifier (must exist in sector file)
//   - RadiusNM: Radius in nautical miles
//   - SpacingDegrees: Vertex spacing (smaller = smoother, but more points)
//
// RECOMMENDED SPACING: 10° for circles up to 20 NM, 5° for larger circles
// ----------------------------------------------------------------------------

// CYHZ_TWR - Halifax Tower (7 NM)
// Activates when: HZT online
// MAP:2:CYHZ_TWR
// COLOR:INTERNAL_DELEGATION
// ACTIVE:CYHZ_TWR:HZT
// COORDPOLY:
// COORD_CIRCLE:CYHZ:7:10
// END

// CYYR_TWR - Goose Bay Tower (10 NM - LARGER than standard!)
// Activates when: YRT online
// MAP:2:CYYR_TWR
// COLOR:INTERNAL_DELEGATION
// ACTIVE:CYYR_TWR:YRT
// COORDPOLY:
// COORD_CIRCLE:CYYR:10:10
// END

// LFVP_TWR - St-Pierre Tower (5 NM - SMALLER than standard!)
// Activates when: VT online
// MAP:2:LFVP_TWR
// COLOR:INTERNAL_DELEGATION
// ACTIVE:LFVP_TWR:VT
// COORDPOLY:
// COORD_CIRCLE:LFVP:5:10
// END

// CYQM_TWR - Moncton Tower (7 NM - standard)
// Activates when: QMT online

// COLOR:INTERNAL_DELEGATION
// ACTIVE:CYQM_TWR:QMT
// COORDPOLY:
// COORD_CIRCLE:CYQM:7:10
// END
